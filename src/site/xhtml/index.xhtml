erel Maven Plugin

Ce plugin a été écrit pour gérer le cycle de vie des projets java de erel, leur inclusion dans la branche de développement, la livraison et le patching

Mise en route

pour commencer à utiliser le plugin, il faut apporter des modifications à votre fichier settings.xml. Vous pouvez prendre exemple sur le fichier settings.xml donné ici, ou simplement utiliser le goal erel:setup qui vous posera des questions afin de configurer votre environement.

Détails de Goals

  • developpement d'un nouvelle fonctionnalité
    • erel:checkout permet de récupérer le projet sur la branche par défaut à partir d'une liste
    • erel:feature-branch permet de créer une feature branche de façon standard. Si la feature branch est créer à partir de develop, c'est une nouvelle fonctionnalité, sinon c'est un hotfix
    • erel:merge-request permet de demander à l'intégrateur de repporter les modifications d'une feature branche sur la branche develop avec revue de code
  • release d'une version planifiée
    • erel:promote permet à la personne en charge du déploiement de qualifier une release candidate en version final
    • erel:release-nextpermet de générer une nouvelle release candidate à partir d'un release branch
    • erel:release-rc1permet de créer une nouvelle release branch à partir de la branche develop, génère la première release candidate (RC)
  • release d'un hotfix
    • erel:release-hotfixPermet de releaser un hotfix dont la merge request a été acceptée
  • gestion de projet
    • erel:add-issuePermet d'ajouter un ticket au changelog
    • erel:setupPermet de configurer le plugin
    • erel:setupPermet de mettre à jour automatiquement les dépendances d'un projet

Pour obtenir de l'aide, utilisez le Help Mojo

mvn erel:help -Ddetail=true -Dgoal= 	<goal-name>
