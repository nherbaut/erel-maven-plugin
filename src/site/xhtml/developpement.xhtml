Les Goals de développement

Coup d'oeil sur les Goals

  • erel:checkout permet de récupérer une copie locale d'un projet erel
  • erel:feature-branch permet de créer une feature branch à partir d'un ticket
  • erel:merge-request permet de soumettre la branche courrante pour une réintégration dans la branche d'intégration

erel:checkout

Permer de récupérer le project sur le repository gitlab sur la bonne branche, à jour, automatiquement à partir d'une liste de choix

  • mvn erel:checkout
  • available-namespace
  • :3 (n.herbaut)
  • :7 (Integration)
  • : pick-a-namespace (3/7): 3
  • : 20 (maven-erel-plugin)
  • : 34 (dummy)
  • : 35 (deployement-tools)
  • : choisissez un projet à brancher
  • checkout terminé avec succès
  • : utilisez mvn erel:feature-branch pour créer une feature branche

erel:feature-branch

Permer de créer intéractivement une feature branch sur le projet que vous venez de récupérer

  • >mvn erel:feature-branch
  • %Merci d'indiquer votre numéro de ticket : 845
  • %Merci d'indiquer le titre du ticket : nouvelle fonctionnalite ccmo
  • %Switched to branch '845-nouvelle-fonctionnalite-ccmo'
  • %Everything up-to-date

erel:merge-request

Permet de soumettre votre travail à l'intégration

  • >mvn erel:merge-request
  • %merci d'écrire un commentaire pour votre requète: merci d'intégrer le cardstore
  • %êtes vous sûr de vouloir soumettre votre requète? (y/n): y
  • %votre requète a été envoyée

en cas de besoin, on peut éviter de passer par des RC pour générer un hotfix. Il suffit d'effectuer le feature-branch sur master et plus sur develop. toutes les autres opérations restent les mêmes (merge request...)

  • >git checkout master
  • mvn erel:feature-branch
